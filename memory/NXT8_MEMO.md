**Версия:** 1.0
**Создан:** 2026-02-05
**Статус:** обязателен к соблюдению при любом изменении `agents/persona_prompts.py`

---

## 1. Жёсткие принципы (не нарушать)

1. **Не ломать существующую логику.** Routing (через `classifier` + `hermes_graph_v2` + `inter_agent.delegate_to_agent`) и набор tool-вызовов агентов остаются как есть. Промпты влияют ТОЛЬКО на качество ответа.
2. **Не менять routing и tool-вызовы.** Если агент сейчас умеет `search_memory`/`create_task`/`web_search` — список `allowed_tools` в `personas.py` не трогаем. Промпт может научить ЛУЧШЕ их использовать, но не добавлять новые.
3. **Только улучшать quality ответов.** Цель промпта: довести агента с уровня джуниора до уровня senior-консультанта по специальности.
4. **Каждый агент остаётся в своей роли.** Bookkeeper не отвечает на юр.вопросы, Compliance не считает unit-economics — для этого `escalate_to_hermes` и `ask_colleague`.
5. **Язык ответов — русский по умолчанию.** Английский — только если пользователь сам пишет на нём.

## 2. Метод усиления (применять для каждого нового агента)

### 2.1 Few-shot примеры
Каждый промпт обязан содержать **минимум 2 примера**: реальный вопрос клиента → ответ эксперта уровня senior-консультанта. Пример показывает агенту:
- структуру ответа (заголовки, нумерация шагов)
- глубину детализации (статьи законов, формулы, цифры)
- тон (без воды, с конкретикой)
- объём (не короче 300 слов на сложный вопрос)

### 2.2 Профильная экспертиза
Каждый агент содержит:
- **Конкретные фреймворки** своей профессии (не просто «знаю Lominger», а «STAR-интервью с 5 вопросами»)
- **Статьи законов и нормативных актов** с номерами (ст. 346.25 НК РФ, ст. 71 ТК РФ, GDPR Art. 33)
- **Числовые benchmarks** с указанием «benchmark» в скобках, а не «думаю что 5%»
- **Region-aware блок** — что меняется при изменении страны компании

### 2.3 Жёсткие правила в конце
Каждый промпт заканчивается секцией «ЖЁСТКИЕ ПРАВИЛА»:
- Нет статьи закона → не утверждай как факт
- Нет данных в контексте → запроси, не выдумывай
- Подпись/решение/деньги — всегда живой человек, агент готовит draft

## 3. Универсальные требования к ответу любого агента

Все агенты ниже `agent_charter.py` соблюдают:

1. **CHARTER** (общий кодекс): иерархия в команде Hermes, проактивный поиск бизнес-ценности, строгий запрет на выдумывание, источник для каждого факта.
2. **Manifest block** — каждый агент знает свою специальность, права доступа, цепочку командования, авторитет принятия решений (`agents/manifests.py`).
3. **Company context block** — регион, индустрия, регуляции, каналы — инжектится автоматически из `core/company_context.py`.

---

## 4. Формат ответа (контракт для UI)

Большинство агентов структурируют ответ так:
1. **Краткое резюме** (1-2 строки)
2. **Контекст / факты** (с источниками)
3. **Действия** (приоритезированный список с owner / deadline / результат)
4. **Эффект** (измеримая метрика)
5. **💡 Возможность для бизнеса** (1-3 идеи если применимо)
6. **Approval needed** (если есть high-impact действие — пометить)

## 5. Уровневая шкала (для self-assessment при правках)

| Уровень | Признак |
|---|---|
| 1-3 | Общие фразы, мог написать любой |
| 4-6 | Базовые знания специальности, но без конкретики |
| 7-8 | Senior-консультант: фреймворки + цифры + статьи законов |
| **9-10** | **Топ-консультант: глубокие знания + few-shot + edge cases + region-aware** |

**Цель для каждого агента — 9/10.**

## 6. Тесты при каждой правке промпта

После любого изменения `persona_prompts.py` ОБЯЗАТЕЛЬНО прогон тестов на эталонных вопросах:

| Агент | Тест-вопрос |
|---|---|
| Hermes | «Я founder, работаю 14 часов в день, команда 7 человек, всё равно всё на мне» |
| Bookkeeper | «Как оптимизировать налоги при переходе с УСН 6% на ОСНО?» |
| Marketer | «Бюджет 50 000 ₽/мес. B2B SaaS. Как распределить между каналами?» |
| Project-coord | «Проект буксует 3 месяца. Команда 4 человека. Дедлайн через месяц» |
| HR-mentor | «Ключевой сотрудник уходит — конкуренты предложили на 40% больше» |
| Client-manager | «Клиент молчит 45 дней. Раньше платил стабильно 8 месяцев» |
| Analyst | «Продажи упали 30% за месяц. Не понимаю почему» |
| Compliance | «Подрядчик не выполнил работу по договору. Что делать?» |

**Критерий пройденного теста:** ответ содержит конкретные цифры/статьи/имена фреймворков, ≥ 300 слов на сложный вопрос, заканчивается actionable шагами с дедлайнами.

## 7. Что хранится где

- `agents/persona_prompts.py` — **deep v2.0 промпты** (главное место правок)
- `agents/personas.py` — конфиг персон (тулзы, fetchers) + bootstrap-fallback промпт
- `agents/agent_charter.py` — общий кодекс
- `agents/manifests.py` — права/иерархия (injected в каждый промпт)
- `agents/ai_mentor.py` — уровневые промпты HR-Ментора (0-1 / 2-3 / 4-5)
- `core/company_context.py` — региональный контекст
